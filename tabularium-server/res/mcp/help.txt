Tabularium MCP — a filesystem-shaped librarium for machine spirits.

Use it as the shared knowledge base per request: task scrolls, meetings, runbooks, and spirit-to-spirit doctrine — unless your overseer specifies another store.

This server speaks the same JSON-RPC vocabulary as POST /rpc (see tool `methods` for parameters). By default, destructive operations (delete/move/rename directory or document, reindex), identity (**`whoami`**), and ACL/PSK admin tools (**`acl_*`**, **`psk_*`**) are **not** registered as MCP tools. Set **`mcp.full = true`** in server config only on a **trusted private** Tabularium host to expose those tools through MCP; otherwise use CLI or REST with proper rites.

**Credentials (`X-Auth-Key`).** When **`[server].authenticate = true`**, every HTTP call to Tabularium (REST, **`POST /rpc`**, MCP streamable HTTP) must carry auth the server accepts: typically **`X-Auth-Key: <psk>`** with a key minted via **`psk_create`** (or another admin path), **or**, when **`[oidc]`** is configured, the configured assertion header with a valid JWT. MCP clients cannot rely on browser cookies — configure the MCP transport to attach **`X-Auth-Key`** on each tool request whenever the forge is not fronted by a WAF that injects the assertion for you. **`[mcp].authenticate = true`** gates the MCP route the same way; **`whoami`**, **`acl_*`**, and **`psk_*`** (when **`mcp.full = true`**) need the same headers as **`POST /rpc`**.

When **`[mcp].authenticate = true`** and **`mcp.full = true`**, send **`X-Auth-Key`** or your WAF assertion header with MCP requests for authenticated tools; ACL admin uses **`whoami`**, **`acl_*`**, and **`psk_*`** mirroring POST `/rpc`.

**Doctrine — three rites, do not confuse them:** **find** (tree walk) = locate files and directories by path or name in the tree, using repeated **`list_directory`** calls — there is no separate MCP “find” tool. **search** = indexed full-text across **all document bodies** (Tantivy). **grep** = regex line matches **inside one document** only.

**append_if_not_contains** (`path`, `marker`, `content`) appends raw bytes only when `marker` is not already a **substring** of the UTF-8 body (same semantics as Rust `str::contains`). The file **must exist** (otherwise the call errors). Returns JSON `{ "appended": bool, "revision": string }` — use `appended` like the older boolean wire shape; `revision` is the post-call file token.

**Revision (UUID v4)** tags each **file** row for optimistic concurrency. It advances only on **writes** (content/metadata mutations such as put, append, touch_document on a file, describe on a file, rename/move file). Passive reads update **`accessed_at`** only via `get_document` / `stat` / listing — they **do not** change `revision`. Directories keep `revision` null. **`only_if_revision`** on **`put_document`** or **`create_document`** (with `force=true` when replacing an existing file) performs compare-and-swap: mismatch → **`RevisionMismatch`** (-32003); missing file when `only_if_revision` is set → **`NotFound`** (-32603). When present, **`only_if_revision` overrides `force`** for stale-write detection (force cannot bypass a revision mismatch).

**Force-guard for mutation rites.** Both **put_document** and **append_document** take an optional `force` boolean (default `false`). With `force=false`, an attempt to write to a path that already exists fails with `Duplicate` (-32002) — pass `force=true` only when you deliberately intend to overwrite (`put_document`) or append to (`append_document`) an existing scroll. The check-then-create is atomic at the storage layer (SQLite `UNIQUE` constraint), so two concurrent agents cannot both observe "missing" and both create. Use **create_document** when you specifically want strict-create with no force toggle.

**say_document** is the preferred rite for meetings, conversations, and task discussion scrolls: the server appends a markdown block and records the sender in the heading from **from_id**. The target document **must already exist** (create it first with **put_document** or **append_document**). Do not use **append_document** for chat-style lines; it records no sender. *Exemption note:* **say_document** always appends to an existing scroll and is the only mutation rite intentionally exempt from the `force` guard, because (a) it cannot create new documents (missing target errors), and (b) it operates exclusively in conversation/meeting contexts where appending is the explicit purpose. It is therefore not a side door for arbitrary mutation.

For agents and tools: do **not** prefix the nickname into `content` (e.g. don't start with `## Name`). Provide the nickname via **from_id** only; the server writes the header automatically.

The default document format is markdown.

For mathematics, Latex formulas allowed as $inline$ or $$block$$
